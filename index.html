HTML

  • Elements are a type of content on a webpage
  • The following elements are used to add content to pages:
    • Text elements
    • Image elements
    • Header elements
    • List elements
  • The following elements are used to divide the content on a page and to describe the content to browsers and other developers:
    • Meta-related elements
    • Content division elements
    • Sematic elements
  • Overal, elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage
  • The head element contains information about the webpage
  • Within the head element we can use the following metadata tags:
    • Title - defines the title of the webpage. This element is required and is extremely important for Search Engine Optimization (SEO)
    • Style - defines CSS styles associated to the website.
    • Base - defines the base URL for a webpage
    • Link - connects an external resource to the HTML document
    • Meta - defines metadata such as the character set, description, keywords, author, and viewport. (For more information Click Here!
    • script embeds client-side scripts in the HTML document
  • The body element represents the visible content shown to the user

CSS

  • To link an external CSS document to our HTML we must input the following within the head section of the HTML:
    < link href="css/styles.css" rel="stylesheet" / >
  • There are 4 main ways to include CSS in your document:
    1. Elements: HTML tags as selectors (such as body, p, h1, etc.)
    2. Classes: HTML classes as selectors (.titles)
    3. IDs: HTML IDs as selectors (#this-is-a-long-ID)
    4. Descendant selectors: The combination of one or more classes, IDs, or elements, separated by spaces, to indicate a family relationship (.titles h1)
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds the changed files in our working directory to the staging area of Git
  • git commit -m "description of changes made": commits changes to our repository
  • git pull origin main: ensures our current branch is up to date (done before opening a pull request)
  • git push origin branch-name: pushes up all changes made in the branch
  • To update out local main branch once a pull request has been merged we input the following:
    • git checkout main
    • git pull origin main
